Problem 1: Continuous Calculator

This program combines a do-while loop to run repeatedly, a Scanner for user input, and a switch statement to perform calculations.

import java.util.Scanner; public class Calculator { public static void main(String[] args) { Scanner scanner = new Scanner(System.in); char operator; double num1, num2, result; do { System.out.print("Enter operator (+, -, *, /) or 'q' to quit: "); operator = scanner.next().charAt(0); if (operator == 'q') { System.out.println("Exiting calculator. Goodbye!"); break; } System.out.print("Enter two numbers: "); num1 = scanner.nextDouble(); num2 = scanner.nextDouble(); switch (operator) { case '+': result = num1 + num2; System.out.println(num1 + " + " + num2 + " = " + result); break; case '-': result = num1 - num2; System.out.println(num1 + " - " + num2 + " = " + result); break; case '*': result = num1 * num2; System.out.println(num1 + " * " + num2 + " = " + result); break; case '/': if (num2 != 0) { result = num1 / num2; System.out.println(num1 + " / " + num2 + " = " + result); } else { System.out.println("Error: Cannot divide by zero."); } break; default: System.out.println("Error: Invalid operator."); } System.out.println("---------------------------------"); } while (true); // The loop runs forever until we 'break' out of it. scanner.close(); } }

Problem 2: Palindrome String Checker

A palindrome is a word that reads the same forwards and backwards, like "madam" or "racecar". Here are two ways to check for one.

public class PalindromeChecker { // Method 1: The Easy Way (Using StringBuilder) public static boolean isPalindromeEasy(String str) { StringBuilder sb = new StringBuilder(str); String reversedString = sb.reverse().toString(); return str.equals(reversedString); } // Method 2: The Optimal Way (Two Pointers) public static boolean isPalindromeOptimal(String str) { int left = 0; int right = str.length() - 1; while (left < right) { if (str.charAt(left) != str.charAt(right)) { return false; } left++; right--; } return true; } // Main method to test both functions public static void main(String[] args) { String test1 = "racecar"; String test2 = "hello"; System.out.println("--- Testing with '" + test1 + "' ---"); System.out.println("Easy method: " + isPalindromeEasy(test1)); // true System.out.println("Optimal method: " + isPalindromeOptimal(test1)); // true System.out.println("\n--- Testing with '" + test2 + "' ---"); System.out.println("Easy method: " + isPalindromeEasy(test2)); // false System.out.println("Optimal method: " + isPalindromeOptimal(test2)); // false } }

Problem 3: The Danger of Integer Overflow

What happens when you add numbers in an array and the sum gets too big for its data type? This is called an overflow, and it can lead to incorrect results.

An int in Java can hold values up to 2,147,483,647. If you add to this, the value will "wrap around" and become negative.
public class ArraySum { public static void main(String[] args) { // These two numbers will cause an overflow when added int[] numbers = { 2_000_000_000, 2_000_000_000 }; // --- The WRONG Way --- int intSum = 0; for (int num : numbers) { intSum += num; } // The result will be a negative number due to overflow! System.out.println("Sum using int: " + intSum); // --- The RIGHT Way --- // Use a 'long' which can hold a much larger value. long longSum = 0; for (int num : numbers) { longSum += num; } System.out.println("Sum using long: " + longSum); // Correctly prints 4,000,000,000 } }

Problem 4: Count Character Occurrences

This is a classic problem that demonstrates looping through a string and using an if condition to check characters one by one.

public class CharacterCounter { public static int countOccurrences(String text, char target) { int count = 0; for (int i = 0; i < text.length(); i++) { if (text.charAt(i) == target) { count++; } } return count; } public static void main(String[] args) { String sentence = "hello world"; char letterToFind = 'l'; int result = countOccurrences(sentence, letterToFind); System.out.println("The letter '" + letterToFind + "' appears " + result + " times."); // Prints 3 } }

Problem 5: Understanding Nested Loops with Patterns

Nested loops (a loop inside another loop) are fundamental for many problems, especially those involving grids or patterns.

public class Patterns { // Pattern 1: Right-Angled Triangle public static void printTriangle(int rows) { System.out.println("Right-Angled Triangle (" + rows + " rows):"); for (int i = 1; i <= rows; i++) { for (int j = 1; j <= i; j++) { System.out.print("* "); } System.out.println(); } } // Pattern 2: Centered Star Pyramid public static void printPyramid(int rows) { System.out.println("\nCentered Pyramid (" + rows + " rows):"); for (int i = 0; i < rows; i++) { for (int j = 0; j < rows - i - 1; j++) { System.out.print(" "); } for (int k = 0; k < 2 * i + 1; k++) { System.out.print("*"); } System.out.println(); } } public static void main(String[] args) { printTriangle(5); printPyramid(5); } } /* Correct output for main(): Right-Angled Triangle (5 rows): * * * * * * * * * * * * * * * Centered Pyramid (5 rows): * *** ***** ******* ********* */

Problem 6: Finding the Second Maximum in an Array

To solve this, we can iterate through the array while keeping track of the two largest numbers found so far.

public class SecondMax { public static int findSecondMaximum(int[] arr) { if (arr == null || arr.length < 2) { System.out.println("Array must contain at least two elements."); return Integer.MIN_VALUE; // Return a sentinel value } int max = Integer.MIN_VALUE; int secondMax = Integer.MIN_VALUE; for (int num : arr) { if (num > max) { secondMax = max; max = num; } else if (num > secondMax && num != max) { secondMax = num; } } return secondMax; } public static void main(String[] args) { int[] numbers = {10, 5, 8, 20, 15, 12}; int secondLargest = findSecondMaximum(numbers); System.out.println("The second largest number is: " + secondLargest); // Prints 15 } }
